# 개발일지 작성 스킬

## 개요

개발 작업 완료 시 `개발일지/` 폴더에 표준화된 형식의 마크다운 파일을 생성한다.

## 작성 시점

다음 상황에서 반드시 개발일지를 작성한다:

1. **새 기능 개발 완료** - 신규 컴포넌트, API, 서비스 추가
2. **버그 수정 완료** - 에러 해결, 동작 오류 수정
3. **리팩토링 완료** - 코드 구조 개선, 성능 최적화
4. **설정 변경** - 환경변수, 빌드 설정 등 변경

## 파일명 규칙

```
개발일지/YYYY-MM-DD-기능명.md
```

예시:
- `2025-12-26-Slack-웹훅-알림.md`
- `2025-12-24-브리핑-생성-MCP-스킬.md`
- `2025-12-20-종목-로딩-성능-최적화.md`

## 필수 섹션

### 1. 헤더

```markdown
# 개발일지 - {기능명}

**작성 시각**: YYYY-MM-DD
**작업자**: {에이전트 역할} (예: backend-developer, frontend-ui-designer)
```

### 2. 해결하고자 한 문제

작업 배경과 요구사항을 명시한다.

```markdown
## 해결하고자 한 문제

{문제 설명 또는 요구사항}
- 요구사항 1
- 요구사항 2
```

### 3. 해결된 것

완료된 항목을 ✅ 체크마크와 함께 나열한다.

```markdown
## 해결된 것

✅ **{작업 카테고리}** (`파일경로`)
- 세부 구현 내용 1
- 세부 구현 내용 2

✅ **{다른 카테고리}**
- 내용
```

### 4. 해결되지 않은 것

미완료 항목이나 알려진 이슈를 ⚠️ 아이콘으로 표시한다.

```markdown
## 해결되지 않은 것

⚠️ **{이슈 제목}**
- 상세 설명

없음  <!-- 이슈가 없을 경우 -->
```

### 5. 향후 개발을 위한 컨텍스트 정리

다음 개발자(또는 에이전트)가 이어서 작업할 수 있도록 정보를 제공한다.

```markdown
## 향후 개발을 위한 컨텍스트 정리

### 사용법
{코드 예시}

### 파일 구조
{관련 파일 목록}

### 다음 개선 사항
1. 개선점 1
2. 개선점 2
```
