# UI/UX 개선 계획

**작성일**: 2025-12-16
**목적**: 메인 페이지 로딩 속도 개선 및 사용자 경험 향상

---

## 1. 현재 문제점 분석

### 메인 페이지 현재 구조
| 섹션 | API 호출 | 로딩 시간 |
|------|----------|----------|
| Header + 검색바 | 없음 | 즉시 |
| 관심 종목 | 없음 (로컬) | 즉시 |
| 알림 설정 | 없음 (로컬) | 즉시 |
| **TOP 3 비교** | fetchTopNStocks | 캐시 히트: 0.01초 |
| **오늘의 화제 종목** | fetchTrendingStock + fetchStockChart | 캐시 미스: 5초+ |
| **브리핑 달력** | fetchBriefings | 1-2초 |
| **브리핑 히스토리** | 동일 | (위와 공유) |

### 문제점
1. 메인 페이지에서 4개 API를 동시 호출
2. 캐시 미스 시 총 로딩 시간 10초 이상
3. 사용자가 핵심 정보를 보기 전에 모든 데이터 로딩 대기
4. 스크롤 없이 보이지 않는 섹션도 함께 로딩

---

## 2. 개선 방향

### 핵심 원칙
> **"First Meaningful Paint"** - 사용자가 가장 중요한 정보를 가장 빨리 볼 수 있도록

### 정보 우선순위 분석
| 우선순위 | 정보 | 이유 |
|---------|------|------|
| 1 (필수) | TOP 3 화제 종목 | 서비스 핵심 가치 |
| 2 (중요) | 종목 검색 | 빠른 접근성 |
| 3 (보조) | 관심 종목 | 개인화 기능 |
| 4 (부가) | 브리핑 히스토리 | 과거 기록 조회 |
| 5 (부가) | 알림 설정 | 설정 기능 |
| 6 (부가) | 브리핑 달력 | 상세 조회 기능 |

## 3. 개선된 페이지 구조

### 3.1 메인 페이지 (/) - 핵심 정보만
```
┌─────────────────────────────────────────┐
│  Header (로고 + 네비게이션)              │
├─────────────────────────────────────────┤
│  종목 검색바                             │
├─────────────────────────────────────────┤
│  🔥 오늘의 화제 종목 TOP 3               │
│  ┌─────┐ ┌─────┐ ┌─────┐               │
│  │ 1위 │ │ 2위 │ │ 3위 │               │
│  └─────┘ └─────┘ └─────┘               │
├─────────────────────────────────────────┤
│  📌 관심 종목 (최대 5개 미리보기)         │
│  [더보기 →]                              │
├─────────────────────────────────────────┤
│  📋 최근 브리핑 (최대 3개 미리보기)       │
│  [전체보기 →]                            │
```

## 4. 네비게이션 개선

### 현재
- 모든 기능이 메인 페이지에 집중

### 개선 후
```
┌──────────────────────────────────────────────────┐
│  🌙 당신이 잠든 사이    [홈] [브리핑] [알림] [⚙️] │
└──────────────────────────────────────────────────┘
```

### 네비게이션 메뉴
| 메뉴 | 경로 | 설명 |
|------|------|------|
| 홈 | / | TOP 3 + 미리보기 |
| 브리핑 | /briefings | 달력 + 히스토리 |
| 알림 | /alerts | 알림 설정 + 히스토리 |
| 설정 | (모달) | 테마, PWA 등 |

## 5. 로딩 UX 개선

### 5.1 스켈레톤 UI
```tsx
// 로딩 중 표시
<div className="animate-pulse">
  <div className="h-32 bg-gray-200 rounded-lg"></div>
</div>
```

### 5.2 점진적 로딩 (Progressive Loading)
1. 먼저 캐시된 TOP 3 표시 (즉시)
2. 관심 종목 로컬 데이터 표시 (즉시)
3. 브리핑 미리보기 지연 로딩 (백그라운드)

### 5.3 Lazy Loading
- 스크롤 시 컴포넌트 로드
- Intersection Observer 활용

## 9. 사용자 시나리오

### 시나리오 1: 빠른 확인
> "오늘 화제 종목이 뭐지?"
1. 메인 페이지 접속 (0.5초)
2. TOP 3 즉시 확인
3. 관심 종목 클릭으로 상세 이동

### 시나리오 2: 과거 브리핑 조회
> "지난주 브리핑 보고 싶어"
1. 메인 → 브리핑 메뉴 클릭
2. 달력에서 날짜 선택
3. 해당 브리핑 상세 조회

### 시나리오 3: 알림 설정
> "TSLA 10% 오르면 알림 받고 싶어"
1. 메인 → 알림 메뉴 클릭
2. 새 알림 추가
3. 조건 설정 후 저장
