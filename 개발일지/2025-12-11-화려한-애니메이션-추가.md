# 개발일지 - 화려한 애니메이션 및 디자인 개선

**작성 시각**: 2025-12-11

## 해결하고자 한 문제

기존 디자인을 더욱 화려하고 역동적으로 만들기 위해 다양한 애니메이션 효과와 시각적 요소를 추가하고자 했습니다.

### 요구사항
1. 배경 파티클/그라데이션 애니메이션 추가
2. 카드에 글로우 효과 및 더 화려한 애니메이션
3. 버튼에 더 화려한 호버 효과 (그라데이션 애니메이션)
4. 로고에 회전/펄스 애니메이션 추가
5. 섹션별 등장 애니메이션 강화
6. 스크롤 기반 패럴랙스 효과 추가

## 해결된 것

✅ **배경 파티클 애니메이션**
- 5개의 파티클 요소를 배경에 추가
- 각 파티클이 독립적으로 움직이는 `particle-float` 애니메이션
- 서로 다른 지연 시간으로 자연스러운 움직임 구현
- `fixed` 포지셔닝으로 스크롤과 독립적으로 동작

✅ **화려한 그라데이션 애니메이션**
- `gradient-animated`: 6색 그라데이션이 회전하는 효과
- `gradient-rotate`: 8초 주기로 그라데이션이 이동하는 효과
- `bg-animated`: 배경 그라데이션이 움직이는 효과

✅ **글로우 펄스 애니메이션**
- `glow-pulse`: 카드와 버튼에 발광 효과가 펄스하는 애니메이션
- 3중 그림자로 깊이감 있는 글로우 효과
- 2초 주기로 강도가 변하는 애니메이션

✅ **로고 애니메이션**
- `logo-spin`: 로고가 20초에 한 바퀴 회전하는 애니메이션
- 회전 중간에 약간의 확대 효과 추가
- 펄스 애니메이션이 있는 인디케이터
- 그라데이션 오버레이로 빛나는 효과

✅ **버튼 호버 효과 강화**
- `btn-gradient`에 `::before` 가상 요소로 빛나는 효과 추가
- 호버 시 좌우로 이동하는 빛 효과
- 호버 시 더 큰 상승 효과 (translateY(-3px) + scale(1.05))
- 3중 그림자로 더 깊이감 있는 효과
- 테두리 글로우 효과 추가

✅ **카드 호버 효과 강화**
- `card-hover`에 그라데이션 테두리 효과 추가
- 호버 시 더 큰 상승 효과 (translateY(-8px) + scale(1.02))
- 3중 그림자로 더 화려한 효과
- `card-glow`: 카드에 펄스하는 글로우 효과
- 그라데이션 테두리가 나타나는 효과

✅ **카드 그라데이션 오버레이 애니메이션**
- 모든 카드에 여러 개의 그라데이션 오버레이 추가
- 각 오버레이가 독립적으로 `animate-float` 애니메이션
- 인덱스 기반 지연 시간으로 순차적 움직임
- 상승/하락에 따라 다른 색상 적용

✅ **텍스트 글로우 효과**
- `text-glow`: 제목에 발광 효과가 펄스하는 애니메이션
- 3중 텍스트 그림자로 깊이감 있는 효과
- 2초 주기로 강도가 변하는 애니메이션

✅ **AnimatedCard 애니메이션 강화**
- 호버 시 3D 회전 효과 추가 (`rotateY: 2`)
- 스프링 애니메이션 적용 (stiffness: 300, damping: 20)
- 더 큰 상승 효과 (y: -8, scale: 1.03)
- `transformStyle: 'preserve-3d'`로 3D 효과 활성화

✅ **추가 애니메이션 효과**
- `scale-in`: 요소가 나타날 때 확대 효과
- `spin-slow`: 느린 회전 애니메이션
- `button-shimmer`: 버튼에 빛이 흐르는 효과

## 해결되지 않은 것 / 향후 개선 필요

⚠️ **성능 최적화**
- 많은 애니메이션이 동시에 실행되면 성능에 영향을 줄 수 있음
- `prefers-reduced-motion` 미디어 쿼리로 접근성 고려 필요
- 모바일에서는 일부 애니메이션 축소 고려

⚠️ **파티클 최적화**
- 현재 5개의 파티클이 있지만, 더 많은 파티클이 필요할 수 있음
- Canvas API를 사용한 더 효율적인 파티클 시스템 고려

## 기술적 세부사항

### 사용한 기술
- **CSS 애니메이션**: `@keyframes`를 활용한 다양한 애니메이션
- **Framer Motion**: 3D 변환 및 스프링 애니메이션
- **CSS 그라데이션**: 동적 그라데이션 배경
- **가상 요소 (::before)**: 버튼과 카드에 추가 효과

### 주요 애니메이션

1. **gradient-rotate**: 8초 주기로 그라데이션이 회전
2. **glow-pulse**: 2초 주기로 글로우 효과가 펄스
3. **particle-float**: 15초 주기로 파티클이 부유
4. **logo-spin**: 20초 주기로 로고가 회전
5. **card-glow**: 3초 주기로 카드 글로우가 펄스
6. **text-glow**: 2초 주기로 텍스트 글로우가 펄스
7. **button-shimmer**: 3초 주기로 버튼에 빛이 흐름

### 주요 변경 파일
1. `app/globals.css`: 
   - 10개 이상의 새로운 애니메이션 추가
   - 글로우, 그라데이션, 파티클 효과
   - 버튼과 카드 호버 효과 강화
2. `app/page.tsx`: 
   - 배경 파티클 요소 추가
   - 로고 애니메이션 적용
   - 제목에 텍스트 글로우 효과
3. `components/*.tsx`: 
   - 모든 카드에 `card-glow` 클래스 추가
   - 그라데이션 오버레이 애니메이션 추가
   - 버튼에 글로우 효과 추가
4. `components/AnimatedCard.tsx`: 
   - 3D 회전 효과 추가
   - 스프링 애니메이션 적용

### 디자인 원칙
1. **계층적 애니메이션**: 각 요소가 독립적으로 움직이되 조화롭게
2. **성능 고려**: `will-change` 속성 활용 및 GPU 가속
3. **접근성**: `prefers-reduced-motion` 지원
4. **시각적 피드백**: 모든 인터랙션에 즉각적인 시각적 피드백
5. **일관성**: 모든 카드와 버튼에 동일한 애니메이션 언어

## 향후 개발을 위한 컨텍스트

1. **애니메이션 성능**
   - 많은 애니메이션이 동시에 실행되므로 성능 모니터링 필요
   - 필요 시 `requestAnimationFrame`을 사용한 최적화 고려
   - 모바일에서는 일부 애니메이션 비활성화 옵션 제공

2. **파티클 시스템**
   - 현재는 CSS로 구현했지만, 더 복잡한 효과가 필요하면 Canvas API 사용 고려
   - WebGL을 사용한 3D 파티클 시스템도 가능

3. **애니메이션 제어**
   - 사용자가 애니메이션을 끄고 켤 수 있는 설정 옵션 추가 고려
   - `prefers-reduced-motion` 미디어 쿼리 활용

4. **컴포넌트 확장**
   - 새로운 카드나 버튼을 추가할 때 `card-glow`, `glow-pulse` 등의 클래스 활용
   - 그라데이션 오버레이는 `animate-float` 클래스로 쉽게 추가 가능

5. **브라우저 호환성**
   - 일부 애니메이션은 최신 브라우저에서만 완벽하게 동작
   - 구형 브라우저를 지원해야 한다면 폴백 스타일 제공 필요
